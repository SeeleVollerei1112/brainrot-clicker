
# EggyUI 审美规范 - 蛋仔风格指南（帧同步版）

> 本文档供 AI 在 **Plan 阶段**推荐风格方向、**Execute 阶段**选择资源时参考。
> 色彩数值为从预设资源推导的建议值，非官方精确规范。
> 帧同步场景下所有数值必须使用 `math.tofixed()` 转换。

---

## 一、蛋仔风格关键词

蛋仔游戏 UI 以**明亮、圆润、卡通**为基础调性：金色暖调传达积极/奖励感，蓝色冷调传达中性/功能感，整体饱和度高、对比度明确、避免纯黑背景。非默认风格包括**悬疑**（暗色+紫红）和**科技**（深色+青蓝）。

## 四、弹窗分级选型

按内容量和场景选择合适弹窗，**禁止用大弹窗装简单内容**。

| 级别 | 适用场景 | 图片预设 ID | 建议尺寸 | 说明 |
|-----|---------|-----------|---------|------|
| **小型**（确认级） | 简单确认/取消、单条提示、奖励展示 | 16347 | 740x290 | 最常用，蛋仔确认弹窗 |
| **小型**（圆角遮罩） | 需要圆角蒙版效果 | 16348 | 790x340 | 蛋仔圆角弹窗遮罩 |
| **中型**（信息级） | 排行榜、道具详情、多按钮操作 | 16347 | 900x500 | 适当放大弹窗尺寸 |
| **大型**（全屏级） | 日历、详细配置、多页内容 | 14588 | 1600x900 | 渐变半透背景，接近全屏，慎用 |
| **气泡**（轻提示） | 教程引导、NPC 对话、简短提示 | 14588 | 300x150 | 非模态，不阻断操作 |

### 弹窗创建示例（帧同步）

```lua
local UINodes = require("Data.UINodes")
local tf = math.tofixed
local canvas = UINodes["画布0"]

-- 全屏半透遮罩
local mask = GameAPI.create_eui_image_at_position(
    14588, canvas,
    tf(960), tf(540),   -- 屏幕中心
    tf(1920), tf(1080), -- 全屏
    "mask_弹窗背景"
)

-- 弹窗背景（小型确认级）
local popup = GameAPI.create_eui_image_at_position(
    16347, mask,
    tf(960), tf(540),   -- 父节点中心
    tf(740), tf(290),   -- 弹窗尺寸
    "panel_确认弹窗"
)
```

## 五、执行原则清单（Execute 阶段对照）

1. **确认用金色按钮，取消用蓝色按钮**
   确认/主操作：按钮预设 `11001`（金色按钮）；取消/次操作：按钮预设 `11002`（蓝色按钮）。严禁反过来使用。

   ```lua
   local tf = math.tofixed
   -- 确认按钮（金色）
   local btn_ok = GameAPI.create_eui_button_at_position(
       11001, parent,
       tf(400), tf(80),
       tf(200), tf(80),
       "btn_确认"
   )
   -- 取消按钮（蓝色）
   local btn_cancel = GameAPI.create_eui_button_at_position(
       11002, parent,
       tf(560), tf(80),
       tf(200), tf(80),
       "btn_取消"
   )
   ```

2. **全屏遮罩统一用 14588**
   ```lua
   local tf = math.tofixed
   -- 正确写法：渐变半透背景
   local mask = GameAPI.create_eui_image_at_position(
       14588, canvas,
       tf(960), tf(540),
       tf(1920), tf(1080),
       "mask_全屏"
   )
   -- 禁止用纯黑不透明背景，会完全遮挡游戏画面
   ```

3. **装饰素材仅在奖励/庆祝场景使用**
   彩带、星星、烟花等装饰性预设只在成功/通关/活动界面添加，日常功能界面不用。

4. **道具必须搭配品质底板**
   展示道具时，底图优先使用圆形底图预设（如 11084 纯色圆形底图），不直接将道具图标放在裸白背景上。

5. **非默认风格（悬疑/科技）仅按场景触发**
   只有当需求关键词明确包含"悬疑/推理/密室/剧本"或"科技/太空/机甲/未来感"时，才使用对应主题资源。其他场景一律使用默认金蓝体系。

6. **文字大小分级参考**
   帧同步中通过 `create_eui_label_at_position` 创建文本后，用 `GameAPI.set_eui_label_font_size` 设置字号：

   | 层级 | 建议字号 | 场景 |
   |-----|---------|------|
   | 主标题 | 40~48 | 弹窗标题、界面标题 |
   | 副标题/按钮文字 | 32~36 | 按钮文案、分区标题 |
   | 正文/说明 | 24~28 | 说明文字、列表条目 |
   | 角标/小字 | 18~22 | 时间戳、版本号、辅助标注 |

7. **控件间距不得小于 20px，内容区留白不得小于 30px**
   - 同类控件（如列表条目）之间垂直间距 ≥ 20px，避免行间拥挤
   - 弹窗内容区上下内边距 ≥ 30px，内容不得贴着弹窗边缘
   - 按钮与内容区底部之间留白 ≥ 40px
   - 实现方式：通过调整节点坐标的 y 值来控制间距，所有间距值使用 `math.tofixed()` 转换

8. **浅色弹窗背景上的文字必须用深色**
   - 正文用深灰 (50, 50, 60, 255)，**禁止在白色/奶白背景上使用白色文字**
   - 次要说明用中灰 (100, 100, 115, 255)
   - 白色文字 (255, 255, 255, 255) 仅用于：按钮文字、深色背景上的文字、悬疑/科技主题深色面板
   - 判断依据：背景亮度高（弹窗白底/奶白/浅蓝）→ 用深色文字；背景亮度低（深色面板/遮罩）→ 用白色文字
