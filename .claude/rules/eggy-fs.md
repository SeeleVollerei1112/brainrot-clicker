# 蛋仔派对帧同步地图开发指引

## 核心原则

1. 严格遵守红线要求：绝不做不能或者用户不想做的。
2. 用户核心需求主导：根据用户需求，指定最佳的实现路径，全力完成用户想要的。
    - 简单需求：干净利落，直接实现。
    - 复杂需求：有条不紊，采用以下流程：需求分析 --> 实现设计 --> 指定计划 --> 实现 --> 自测验证。
3. 高危操作预警：高风险操作（例如大范围改动）主动向用户提出确认，给出建议或者备选方案。

## 基础概念（术语）

- 组件：游戏中的建筑、物品、生物等一系列单位的统称。场景内可以被摆放的东西都是组件。
- 预设：单一的配置数据，可以是针对各种系统的（比如：组件、技能）。预设是分层级的，地图预设 >> 内置预设（官方预设）。

## 开发工具

### 蛋仔派对帧同步编辑器

专为蛋仔派对游戏打造的地图编辑器，分为编辑与试玩两部分：

    - 编辑：用于进行地图内容的编辑创作，包括但不限于：资源管理、场景搭建、玩法逻辑开发等。
    - 试玩：在模拟环境下运行当前正在制作的地图，主要用于体验、调试地图内容，包括但不限于：地图场景、玩法等。

编辑器配有配套的 CLI 工具（ editor-cli.exe ），可以通过工具直接对编辑器下达指令从而操作编辑器，工具默认位于当前工程根目录下的 [.codemaker](../.codemaker) 目录下。

    > 若 [editor-cli.exe](.codemaker/editor-cli.exe) 不存在，请要求用户提供 editor-cli.exe 的完整路径。
