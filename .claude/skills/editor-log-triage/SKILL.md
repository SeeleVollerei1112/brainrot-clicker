# 报错排查指南

## 交互规范

1. **第一句固定回复**：`开始排查。`
2. **读取日志**：直接读项目根目录 `log.txt`
3. **分析输出**：
   - 错误摘要（1~3 条关键 LUA_TRACE）
   - 可能原因
   - 解决方案（推荐 + 备选）
   - 验证步骤

## 输出格式

```
## 错误摘要
1. [关键错误]

## 可能原因
[分析]

## 解决方案
**推荐**：最小修复步骤
**备选**：其他方案

## 验证步骤
[如何确认修复]
```

## 日志不足时

提出 1~2 个关键补充信息请求：
- 相关 lua 文件片段
- 触发步骤
